20. Line and system maps and map repository

Line map repository

One thing that I handled was the line map repository.

There are actually two line map repositories: the one called “Repository” and the one called “Schematics”, both in PubInfSys – Documents/Maps. We stopped doing schematic maps for training purposes after the big budget cuts in 2009 or so, but we still have the old ones.

I described this in excruciating detail in two instruction documents (“Maps Repository – Quick Start” and “Maps Repository – Extended”) back when this was first created. Other than that I removed the creation of JPEG maps for the web (since these are no longer used), these documents still pretty well describe how it works.

But in short: the repositories are just big sets of folders and files, with a naming convention that allows a program to determine which of the line maps are the current ones. The mr_import program renames a file to match the naming convention and moves it to the right place in a folder; the mr_copy program gets the latest version of each map and copies it into a different folder. That’s it.

The truth is that nothing these programs do couldn’t be done manually. Anybody could move a map into a folder with the same name as the lines described by that map, and then copy the current map into another folder when a new one comes in. It’s not even that inefficient, compared to some of the things we do.

It would be possible to install Perl and these programs, and then run the programs on any Mac. There isn’t any access to the database that would make it hard to run on any system. There just needs to be a line in .actium.ini listing the location of the repository. Mine is:

[Repository]

repository = /Users/apriven/p/Maps/Repository

Where “~/p” is a symlink to the “PubInfSys - Documents” directory.

It would be much better, though, if a real digital asset management system were procured, which would allow versions to be better tracked and assets tagged with the appropriate line designations.
