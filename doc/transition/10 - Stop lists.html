10. Stop lists

Making stop lists

Making the stop lists is just a matter of running the program.

After importing the data into the system, run the following program:

actium.pl avl2stoplists -s signup

That makes two sets of text files. In slists/pat are lists of stops for each pattern, and in slists/line, lists of stops for each direction (which is to say, it combines the list of stops for the patterns). Each file has a stop name, plus some other information stored in the FileMaker database (site and direction).

Issues and Future

This is an older program, and uses Hastus Standard AVL Export file format as its input. I wrote a little shim that converts the Enterprise Database Export back into this format, which is run as part of actium.pl newsignup. This works, but is problematic for a couple of reasons. The stop order may not be the same as in the schedule files, which use a newer, better algorithm (although still imperfect) for reassembling stops into their proper order. Moreover, the schedule files can be edited, but these stop lists won’t pick that up.

Probably this function is simple enough that it ought to come from Scheduling rather than from us.

One issue is that Hastus doesn’t keep track of the direction of travel of each stop. It keeps track of the corner (northeast, southeast, northwest, southwest), and if you have the corner and the “near side / far side” information, you can determine the direction. However, it’s not present for many stops that are not at corners. That’s a very useful thing to know.
