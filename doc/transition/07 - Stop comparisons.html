7. Stop comparisons

The process of comparing stops is actually pretty simple. The computer program does most of it.

After importing the data into the system (using the “newsignup” document), run the following program:

actium.pl comparestops -o oldsignup -s newsignup

Where “oldsignup” and “newsignup” are the signup folders for the relevant signups. For example,

actium.pl comparestops -o f22 -s w22

That will leave a file called “comparestops-f22-w22.txt” in the compare folder of the newer signup directory (PubInfSys – Documents/Actium/w22/compare).

This is a simple tab-delimited text file. The first column, “Change,” is a code indicating the type of change: AS = added stop, RS = removed stop, AL = added lines, RL = removed lines, CL = changed lines (lines both added and removed). The other fields are straightforward: the stops ID, the stop description, and the lists of lines added, removed, or unchanged. I usually load this into Excel and make notes on what kind of changes need to happen because of those changes. The process of deciding what to do at each stop – bag it, put a temporary sign on it, etc. – is a manual process of looking at each one and deciding what to do.

One complication is with the 600-series lines. Summer signups it’s usually best to specify the “‑omit” and “-600s” switches, which will tell it not to look at those lines; otherwise it will tell you about every stop that has a 600 series line being taken out, which is frustrating. (Even then you have to be aware of stops that are served only by school-day-only versions of regular lines, like the school-day-only stops on 95 at Hayward High and Bret Harte Middle School.) For Fall signups, one would normally do a comparison between Summer and Fall using both the “-omit” and “-600s” switches, which omits those lines, and then a second comparison between Spring and Fall with just the “-600s” switch, which will then only compare 600 lines. Then those have to be combined.

There is also a program called “actium.pl comparestops3”, which just displays the routes for three different signups. I haven’t used it much.
